HTML Elements

Tag Element Description Block, Inline, N/A Container or Empty
<!--..--> The comment tag is used to insert comments in the source code. They are not displayed in browsers. N/A Container
<!DOCTYPE> Declares the type of HTML being used on the page and allows for browsers to render content correctly. It must be the first thing in your document. N/A Empty
<article> Identifies content that can stand alone seperate from other content. Block Container
<aside> Identifies content that is related to the main content, but is not primary content. Block Container
<body> Identifies a document's body contents. Body Container
<br> Visual element that creates a break in a content line. It is similar to the 'return' key. Inline Empty
<dd> Used to describe a list item. Block Container
<div> Used as a container for holding Javascript, CSS, or code that can't be defined by any other HTML tag. Its use is typically discouraged. Block Container
<dl> Used to describe a list with descriptions. Block Container
<dt> Defines an item/term in a descriptions list. Block Container
<em> Defines content as emphasized. Browsers render emphasized text in italics, but also offers semantic value and accessibility for those using screen readers. Inline Container
<fieldset> Used to group related elements in a form. Block Container
<figure> Specifies self-contained content such as illustrations, diagrams, and photos. Block Container
<figcaption> Defines a caption for a <figure> element. Block Container
<footer> Defines the footer/bottom section of the page that usually contains information like author, copyright, contact information, etc. Block Container
<footer> Defines the footer/bottom section of the page that usually contains information like author, copyright, contact information, etc. Block Container
<form> Used to create an HTML form for user input. Block Container
<h1> to <h6> HTML tags used to define headings. <h1> being the most important heading and <h6> being the least important. Inline Container
<head> Element used as a container for non-visual elements in the document such as styles, scripts, etc. Block Container
<header> Element that represents a container for intro content and/or the page navigation. Block Container
<html> Tag that tells the browser the document is using HTML. Block Container
<img> Defines an image in a HTML document. Block Empty
<iframe> Specifies an inline frame. Block Container
<input> Specifies a input field for users to enter data. Inline Empty
<label> Defines a label for buttons, or checkboxes. It allows for more usability by allowing users to select buttons by making text in the <label> field clickable. Inline Container
<li> Defines a list item. Inline Container
<link> Defines a link between a document and an external resources such as CSS documents. Inline Empty
<meta> Used to provide metadata about an HTML document, such as keywords, last modified and other info for browsers to correctly render the page. It doesn't have visual properties. Inline Empty
<nav> Used to define the navigation of a page. Block Container
<ol> Defines an ordered list. Block Container
<ul> Defines an unordered list. Block Container
<p> Defines a paragraph. Inline Container
<script> Element that contains client-side Javascript or points to an external Javascript file through the src attribute. Block Container
<section> Defines sections in a document. Block Container
<strong> Defines important text. Visually bolds text, but other than the <b> tag, it gives content semantic value. Inline Container
<style> Used to define style information for an HTML document. Block Container
<table> Defines a data table. Block Container
<td> Defines a data cell in an HTML table. Block Container
<tr> Defines a table row in an HTML table. Block Container
<th> Defines an HTML table header Block Container
<title> Required in all HTML documents, as it defines the title of the document and appears in the tab in a web browser. The title will also display in search-engine results and when the web page is favorited. Block Container
